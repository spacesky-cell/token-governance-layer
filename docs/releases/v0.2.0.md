# Token Governance Layer v0.2.0

**DRAFT release notes.** Publication is allowed only after the reproducible
artifact, preflight evidence, and required CI gates pass. The frozen commit SHA
and artifact SHA-256 are inputs to the GitHub draft and npm publication.

## Highlights

- Conservative, evidence-backed governance for registered Claude Code output.
- Project-local Hook and MCP installation with read-only integration diagnosis.
- Payload-free operational evidence and explicit ledger lifecycle controls.
- Hardened standalone MCP lifecycle and experimental tools-only Gateway.
- Deterministic benchmarks and a packed-artifact release verification matrix.

## Compatibility

The package supports Python 3.10 through 3.14 and Node.js 22 or 24. Persistent
Claude installation requires a global npm installation; `npx` and `npm exec`
remain available for one-shot commands only.

## Breaking changes

- Generic head/tail summarization and the old `GovernanceEngine.govern_context`
  compatibility behavior are removed.
- CLI and MCP governance requests use the registered strategy contract.
- MCP original retrieval is bounded and paginated by default.

## Release gate and verification status

Before publication, the controller must verify the clean frozen commit, package
manifest, artifact hash, preflight authorization, and required CI checks. The
GitHub draft/release and npm publish consume that exact frozen SHA and artifact
hash without rebuilding.

After publication, remote evidence verifies GitHub main/tag/release metadata,
npm version/latest/gitHead/integrity, the registry tarball hash, and a clean
registry installation. Remote evidence is post-publication verification, not a
pre-publication gate.
